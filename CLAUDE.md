# CLAUDE.md — 에이전트 실행 지침

> 이 레포는 **위키를 만드는 6가지 방식의 메뉴판**이다.
> 에이전트는 사용자가 고른 **하나의 방식만** 만든다. 전부 만드는 레포가 아니다.
> 사용자가 클론 후 "위키 만들어줘"라고 하면 아래를 따른다.

---

## 흐름 (3단계)

```
[1] 방식 선택 ──▶ [2] 온톨로지 확정 ──▶ [3] 해당 RECIPE.md 실행
     메뉴판          ontology.yaml           recipes/XX/
```

## [1] 방식 선택 — 메뉴판을 보여줘라

사용자에게 아래 표를 제시하고 **하나를 고르게 한다** (조합 원하면 그때 조합):

| # | 방식 | 이런 사람에게 | 조합(표현+연결+접근) |
|---|---|---|---|
| 01 | **Wiki md** | 그래프로 지식 지형을 보고 싶다, 옵시디언 사용자 | 요약 md + 심링크/정션 + 파일읽기 |
| 02 | **SQLite DB** | 정확한 조인·필터·집계가 필요하다 | 레코드 + 외래키 + SQL |
| 03 | **MCP 서버** | 어떤 세션에서든 `/wiki`처럼 검색하고 싶다 | 02의 DB + MCP 툴콜 |
| 04 | **RAG** | 키워드는 모르고 "비슷한 거" 찾고 싶다 | 벡터 청크 + 메타데이터 + 유사도검색 |
| 05 | **RSS/정적사이트** | 남(사람·타 에이전트)이 구독하게 하고 싶다 | 피드 + URL + HTTP |
| 06 | **OKF 번들** | 표준 규격으로 만들어 타 조직·타 도구와 호환하고 싶다 | 요약 md + type frontmatter + md링크 |

사용자가 뭘 고를지 모르겠다고 하면, "어떤 질문을 던질 것인가"를 물어서
질문 형태로 추천한다 (각 RECIPE.md 상단의 '잘 답하는 질문' 참고).

## [2] 온톨로지 확정 — 만들기 전에 뼈대부터

어떤 방식을 골랐든 **`ontology/ontology.yaml`이 공통 입력**이다.

1. `ontology/ontology.yaml`(강의안 예시)과 `ontology/examples/`를 보여준다
2. 사용자의 자료에 맞는지 묻고, 다르면 `ontology/GUIDE.md`의 역산법으로 수정:
   - 실제 검색 문장 3개 → 반복 명사 = 개념/개체, 동사 = 관계
3. 확정본을 레포 루트에 `my-ontology.yaml`로 저장

구조에 정답은 없다. `ontology/GUIDE.md`의 **5가지 패턴**(트리플·계층·패싯·이벤트·최소) 중
자료와 질문 형태에 맞는 것을 사용자와 함께 고른다. 트리플(개념-개체-관계)이 디폴트가 아니다.
그리고 document 개체에는 반드시 `source_path`(원문 포인터)가 있어야 한다.
위키에는 요약만 저장하고, 원문은 링크/경로로 도달한다 — 이것이 깨지면 실패다.

## [3] 실행

선택한 `recipes/XX/RECIPE.md`를 읽고 그대로 수행한다.
모든 레시피는 `my-ontology.yaml`을 읽어 자기 방식으로 물질화한다:

```
같은 온톨로지, 다른 물질화
  01: relations → [[위키링크]]      02: relations → 외래키
  04: 축 → 청크 메타데이터          05: topic → <category>
```

완료 후 검증: 사용자의 검색 문장 3개를 직접 실행해
(a) 요약이 검색되고 (b) source_path로 원문까지 도달하면 성공.

---

## 절대 규칙

1. **한 번에 하나만.** 5개를 다 만들자고 제안하지 마라. 하나를 완성하고,
   조합(예: 02→03, 01→05)은 사용자가 원할 때만.
2. **온톨로지 없이 시작 금지.** 뼈대 확정 전에 파일을 생성하지 마라.
3. **원문 복사 금지, 포인터 필수.** source_path 없는 요약은 실패다.

## 레포 구조

```
CLAUDE.md                  ← 이 파일 (에이전트 진입점)
README.md                  ← 사람용 철학
ontology/
  ontology.yaml            ← 온톨로지 정의 (트리플 패턴·강의안 예시) ★ 공통 입력
  GUIDE.md                 ← 질문→온톨로지 역산법 + 구조 다이어그램
  examples/                ← 패싯(제안서)·이벤트(회의록) 등 타 패턴 예시
templates/                 ← 요약 md frontmatter 등
recipes/01~06/RECIPE.md    ← 방식별 독립 실행 지침 (06 = 구글 OKF 표준 호환)
```
